# Concepts

Maestro has a few core components that work together. Understanding how they fit will help you configure and deploy effectively.

```mermaid
graph TB
  subgraph ORCH ["orchestrator · observes all transitions, triggers each agent"]
    direction TB
    TI["tracker integration"] -. "new task" .-> CC["centcom"]
    USR["user"] -. "sees new task" .-> CC
    USR -. "move status 'in-progress'" .-> IA

    IA["implementation agent"] -. "pushes code, polls for comments,<br/>merge conflicts, pipeline failures" .-> CB[("codebase<br/>remote source")]
    IA -- "loop" --> RA
    RA -- "remains 'in-progress'" --> IA
    RA["review agent"] -. "review MR, post comments,<br/>remains in-progress" .-> CB
    RA -. "in progress" .-> RP["risk profile agent"]

    RP -. "risk assessment" .-> HG{{"human gate<br/>approve & merge"}}
    HG -. "approved" .-> DA["deploy agent<br/>lower envs"]

    QA -. "release to prod" .-> DA2["deploy agent<br/>prod"]
```
