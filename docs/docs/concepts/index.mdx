# Concepts

Maestro has a few core components that work together. Understanding how they fit will help you configure and deploy effectively.

## System overview

Five components, one flow:

```mermaid
graph LR
  USR["User"] -. "creates issue" .-> EXT["Trackers & Code Hosts<br/>GitHub · GitLab · Linear · Jira"]
  EXT -. "syncs issues" .-> CC["CENTCOM<br/>orchestrator + API"]
  CC -. "dispatches" .-> AL["Agentic Loop<br/>implement · review · risk"]
  AL -. "PRs, comments,<br/>CI checks" .-> EXT
  AL -. "approved" .-> DP["Exit<br/>deploy · monitor · done"]
```

## The agentic loop

The core of Maestro — agents implement, review, and iterate until the code is ready.

```mermaid
graph TB
  subgraph LOOP ["agentic loop"]
    direction TB
    IA["implementation agent"] -- "opens PR,<br/>pushes code" --> CB[("codebase<br/>remote source")]
    IA -- "ready for review" --> RA["review agent"]
    RA -- "request changes" --> IA
    RA -. "review MR,<br/>post comments" .-> CB
    RA -- "approved" --> RP["risk profile agent"]
    RP -. "issue found:<br/>comment on diff" .-> IA
  end

  RP -- "risk assessment" --> OUT["human gate / auto-approve"]

  style LOOP fill:none,stroke:#b45309,stroke-width:2px,stroke-dasharray:10 5,color:#b45309
```

## CENTCOM and workers


## Exit: deploy and monitor

After the agentic loop approves, the task exits through deployment and monitoring.

```mermaid
graph TB
  HG{{"human gate<br/>approve & merge"}} -. "approved" .-> DA["deploy agent<br/>lower envs"]
  DA -. "test in staging" .-> QA["QA agent<br/>verify staging"]
  QA -. "issues found" .-> BACK["back to agentic loop"]
  QA -. "all clear" .-> DA2["deploy agent<br/>prod"]
  DA2 -. "watch 15 min" .-> MA["monitor agent"]
  MA -. "anomaly detected" .-> BACK
  MA -. "all clear" .-> DONE["done"]

  style HG fill:#ebe5d9,stroke:#2c2416,stroke-width:2.5px,color:#2c2416
```
